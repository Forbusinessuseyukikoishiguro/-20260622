> 完璧再現を前提に、設計判断・トレードオフ・拡張ポイントを簡潔に記す。冗長な説明は省く。

## 1. 要件定義

- **目的**：PoC現場の認識齟齬・スコープ膨張・「言った言わない」を、仕組みで抑止する。
- **機能要件**：10機能（①〜⑩）。詳細は §3 表。
- **非機能要件**：単一HTML／ビルドレス／オフラインファースト（AI除く）／ローカル永続化／a11y（aria・キーボード・reduced-motion）／出典非捏造。
- **制約**：claude.aiアーティファクト実行を一級市民とする（`window.storage`・`api.anthropic.com` 前提）。同一ファイルがローカルHTMLとしても動く。

---

## 2. アーキテクチャ方針

| 決定 | 根拠 | トレードオフ |
|------|------|--------------|
| 単一HTML＋vanilla JS | 配布・実行コストゼロ、レビュー容易 | 大規模化で1ファイルが肥大（モジュール境界で緩和） |
| 集中スナップショット状態 | 保存/復元/版管理を一括化 | グローバル状態。命名規約とモジュール分割で統制 |
| Storage抽象（3段フォールバック） | 同一コードで artifact/local/memory を吸収 | 同期/非同期差をasyncで統一（軽微なオーバーヘッド） |
| 「データ→render」単方向 | 再描画で整合性担保、差分管理不要 | 粒度の粗い再描画（規模的に許容） |
| AIは加法的機能 | オフライン基盤を毀損しない | AI依存機能はオンライン限定（明示） |
| 出典非捏造をsystemに固定 | 信頼性・コンプライアンス | 回答が控えめになる場合あり（許容） |

### 3-1. モジュール一覧

| ① | フェーズ＋危険サイン | `PHASES` / `renderPhaseDetail` | （操作のみ） |

| ③ | 和風いいかえ | `IIKAE` / `renderIikae` / `aiIikae` | — |
| ④ | 確認メモ | `buildMemo*` / print・CSV | `snapshot.memo` |
| ⑤ | AI壁打ち（梅） | `sendToSenpai` / `attachSpeech` | （会話は非永続） |
| ⑥ | タスク・報連相 | `tasks` / `renderBoard` | `snapshot.tasks` |

横断：`Store`（永続化）、`renderKoto`＋`kotoPresets`（AI/音声パラメーター＋プリセット）、同意ゲート＋`consentLog`、`snapshotState/applyState`、扇子UI。

### 3-2. Storage抽象

```
Store.detect()  // artifact > local > memory
Store.get/set/del/listKeys  // 全async、JSON直列化
```
- presets/consentはスナップショットと別キー（`usa:kotopresets` / `usa:consentlog` / `usa:consent`）で独立永続化。

### 3-3. 状態モデル（snapshot v2.8）

```
{ v, savedAt, log, tasks, kick, ume{history,view,mode,doc},
  conc{history,view,channel}, learnNotes,
  saku{history,view,req}, phrases, koto }
```
- AI会話は「`history`（API用・doc同梱あり）」と「`view`（表示用・素の入力）」を分離。表示クリーンさとAPI文脈を両立。

### 3-4. AI統合

```
callClaude({system, history, tools, maxTokens})
  body = { model:"claude-sonnet-4-6", max_tokens: koto.len,
           temperature: koto.temp, system, messages, tools? }
  POST https://api.anthropic.com/v1/messages
```
- `collectText()` で text ブロックのみ連結（tool_use/result は無視）。
- 論文参照は `web_search_20250305` を tools に付与。
- system に「査読論文＋参照先明記＋捏造禁止」を固定。⑩は加えて「王道／王道以外の2系統」を強制。

### 3-5. 音声

- `attachSpeech(btnId,targetId,heardId,statusId)`：`SpeechRecognition` をボタンへ束縛。
- `onresult` で `isFinal` を分離 → `.final`（確定）/`.interim`（途中）で色分け。
- 自動停止は `koto.voiceWait` 秒。非対応環境はボタン無効化＋テキスト誘導。`demoRecognition()` で色分けを実機相当にデモ。

## 4. 処理フロー（主要パス）

1. **起動**：`initGate` → consent判定（保存）→ `bootAfterConsent` → `buildApp` → autosave復元。
2. **依頼書→④/⑦**：`reqVals` → `reqToMemo`/`reqToKick`（⑦は`updateKLine`再生成）。
3. **AI相談**：`*Send` → `callClaude`（+web_search）→ `collectText` → 描画 → `view/history` push → `persistAuto`。
4. **保存連携**：⑨/⑩の直近回答 → `*ToLog`/`*ToTask`（②`logEntries`／⑥`tasks`へ）。

## 5. テスト戦略

- **静的**：インラインJS抽出 → `node --check`。
- **動的**：`jsdom` で実行。`fetch`/`print`/`a.click`/`isSecureContext` をスタブ化。状態遷移・保存/復元・API引数（tools/system/messages）をアサート。
- **音声**：SR非対応のため色分けはデモ機能で代替検証（既知の限界）。
- **回帰**：各版で独立スイート（v2.1〜2.8、合計 240+ ケース、全緑）。v2.8単体：**30/30 pass**。

## 7. リスクと対策

| リスク | 対策 |
|--------|------|
| 単一ファイル肥大 | モジュール境界の明確化、データ駆動化 |
| API非接続/失敗 | try/catchで降格、非AI機能は無影響 |
| ストレージ不可環境 | memoryモード＋エクスポート導線 |
| 出典の誤り | system固定＋web_search＋人手確認前提の明示 |
